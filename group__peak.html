Detailed Description

Support for peak-finding plots.

The basic concept behind peak-finding is described in [1], although this implementation is based on an older version of Cody et al.'s algorithm.

Calculates the number of monotonic intervals in a light curve having a magnitude change greater than a threshold.

Parameters
[in]timesTimes at which data were taken
[in]dataThe data (typically fluxes or magnitudes) measured at each time
[in]minAmpThe smallest change in the value of data to count
[out]peakTimesThe times of the endpoints of significant intervals.
[out]peakHeightsThe heights of the endpoints of significant intervals.
Precondition
times contains at least two unique values
times is sorted in ascending order
data.size() = times.size()
data[i] is the measurement taken at times[i], for all i
neither times nor data contains NaNs
minAmp > 0
Postcondition
the data previously in peakTimes and peakData are erased
1 ≤ peakTimes.size() = peakData.size() ≤ times.size()
for all i in peakTimes except the first and last elements, (peakTimes[i], peakData[i]) are the coordinates of a local minimum or maximum in (times, data)
for all i in peakTimes except the first two and last single elements, if peakTimes[i] is a local minimum in (times, data), then peakTimes[i-1] is a local maximum, and if peakTimes[i] is a local maximum, then peakTimes[i-1] is a local minimum
for all i in peakTimes except the first element, peakHeights[i] differs from peakHeights[i-1] by at least minAmp
for all points in (times, data) that are not in (peakTimes, peakHeights), inserting that point anywhere in (peakTimes, peakHeights) would violate one of the above conditions
Performance
O(N) time, where N = times.size()
Exceptions
kpftimes::except::BadLightCurveThrown if times and data do not have at least two values.
std::invalid_argumentThrown if times and data do not have the same length or if minAmp is not positive.
std::bad_allocThrown if there is not enough memory to find the peaks
Exception Guarantee
The function parameters are unchanged in the event of an exception.

Calculates the waiting time for variability of a given amplitude as a function of amplitude.

Parameters
[in]timesTimes at which data were taken
[in]dataThe data (typically fluxes or magnitudes) measured at each time
[in]magCutsThe variability amplitudes at which to characterize the timescale
[out]timescalesThe waiting times
Precondition
times contains at least two unique values
times is sorted in ascending order
data.size() = times.size()
data[i] is the measurement taken at times[i], for all i
neither times nor data contains NaNs
magCuts[i] > 0, for all i
Postcondition
the data previously in timescales is erased
timescales.size() = magCuts.size()
timescales[i] is the waiting time for variability greater than magCuts[i], for all i
Performance
O(CN log N) time, where C = magCuts.size() and N = times.size()
Exceptions
kpftimes::except::BadLightCurveThrown if times and data do not have at least two values.
std::invalid_argumentThrown if times and data do not have the same length or if minAmp contains negative values.
std::bad_allocThrown if there is not enough memory to compute the timescales
Exception Guarantee
The function parameters are unchanged in the event of an exception.
