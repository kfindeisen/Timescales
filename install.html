Installation

Requirements

Timescales is written in standard C++, and should build on any C++98/C++03-compliant compiler. For portability between build environments, no TR1 or C++11 features are included. However, the makefile itself is specific to GCC, so users wishing to install Timescales on a non-Unix platform may need to revise the contents of the following files:

  • makefile
  • makefile.inc
  • makefile.common
  • makefile.subdirs

The library may switch to CMake in the future for improved portability.

Timescales depends on the following external libraries:

  • Boost 1.33 or later
  • Gnu Scientific Library (GSL) 1.3 or later

In addition, (re-)generating this documentation requires Doxygen 1.8.1 or later.

Build Commands

Running make, with no arguments, will build libtimescales.a in the timescales directory. Running make unittest will build the unit test suite at tests/test. Running make autotest will build the library and the test suite, if neccessary, before running all tests. Finally, running make doc will generate this documentation at doc/html/index.html and (if you have LaTeX installed) doc/latex/refman.pdf, and running make all will generate the library, the test suite, and the documentation.

Place timescales.h in a directory where your C++ compiler can find header files, such as /usr/local/include/. Place libtimescales.a in a directory where your C++ compiler can find static libraries, such as /usr/local/lib. A shared version of the Timescales library is not available at present.

Using Timescales in Your Software

To use Timescales, include <timescales.h> in your source code (see examples/example.cpp). You must link the final program with -lgsl as well as -ltimescales. The library depends on Boost only through template headers, so you do not need to link against any Boost libraries.
